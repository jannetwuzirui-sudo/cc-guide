# 快捷键速查

熟练使用快捷键能让你在 Claude Code 中的操作更加流畅。以下是自由职业日常开发中最实用的 10 个快捷键。

## 核心快捷键一览

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| `Escape` | 取消当前操作 | 中止正在输入的内容，回到空白输入状态 |
| `Ctrl + C` | 中断执行 | 停止 Claude Code 正在进行的任务，比如生成代码或读取文件 |
| `Tab` | 自动补全 | 补全文件路径、命令名称等，减少手动输入 |
| `Up Arrow` | 历史记录上翻 | 快速调出之前输入过的指令，避免重复打字 |
| `Down Arrow` | 历史记录下翻 | 配合上箭头浏览历史指令 |
| `Ctrl + L` | 清屏 | 清除终端显示内容，保持界面整洁（不清除对话上下文） |
| `Ctrl + A` | 光标移到行首 | 快速跳到输入行的开头进行编辑 |
| `Ctrl + E` | 光标移到行尾 | 快速跳到输入行的末尾 |
| `Ctrl + U` | 删除整行 | 清空当前输入的内容，重新开始输入 |
| `Ctrl + W` | 删除前一个单词 | 快速删除刚输入的一个词，比逐字删除更高效 |

## 高效使用技巧

### 1. 善用 Escape 控制节奏

当你发现给出的指令不够准确，Claude Code 开始往错误方向执行时，立刻按 `Escape` 中止，重新组织语言再输入。这比等它执行完再纠正要节省大量时间和 token。

### 2. 用 Up Arrow 复用指令

自由职业中经常需要对不同文件执行类似操作。按 `Up Arrow` 调出上一条指令，稍作修改就能复用，比如：

```
审查 src/api/users.ts 的代码质量
# 按 Up Arrow，修改文件名
审查 src/api/orders.ts 的代码质量
```

### 3. Ctrl + C 是你的安全网

如果 Claude Code 正在执行一个耗时很长的操作（比如分析整个大型项目），而你意识到方向不对，果断按 `Ctrl + C` 中断。已经消耗的 token 无法退回，及时止损很重要。

### 4. Tab 补全提升准确性

输入文件路径时多用 `Tab` 补全，避免手动输入路径出错导致 Claude Code 找不到文件，白白浪费一轮对话。

### 5. 清屏不等于清空

`Ctrl + L` 只是清除屏幕显示，对话上下文仍然保留。当屏幕内容太多影响阅读时放心使用，不会丢失任何信息。如果要真正清空上下文，请使用 `/clear` 命令。

## 日常操作组合

以下是几个常见的操作组合，形成肌肉记忆后效率会大幅提升：

- **快速重试**：`Ctrl + C` → `Up Arrow` → 修改指令 → `Enter`
- **切换任务**：输入 `/clear` → 输入新任务指令
- **整理输入**：`Ctrl + U` 清空当前行 → 重新输入更精确的指令
- **精确定位文件**：输入部分路径 → `Tab` 补全 → 确认后 `Enter`

掌握这些快捷键后，你在 Claude Code 中的操作会变得非常流畅，把更多精力放在思考和决策上，而不是重复的手动操作。
