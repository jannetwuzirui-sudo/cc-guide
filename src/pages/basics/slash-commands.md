# 核心 Slash 命令精选

掌握这 15 个命令，覆盖自由职业开发中 90% 的场景。每个命令都附带实际接单中的用法。

## 项目初始化

### /init

生成 `CLAUDE.md` 项目记忆文件，让 Claude Code 快速了解项目背景。

```
/init
```

**接单场景**：拿到客户项目后第一件事就执行它。自动分析项目结构、技术栈、依赖关系，后续对话效率大幅提升。

---

### /memory

查看和编辑项目记忆文件。

```
/memory
```

**接单场景**：在 CLAUDE.md 中补充客户的特殊要求，比如"代码风格用 Airbnb 规范"、"不要用 any 类型"，让每次对话都自动遵守。

---

## 会话管理

### /compact

压缩当前对话上下文，保留关键信息，释放 token 空间。

```
/compact
```

**接单场景**：开发复杂功能时对话会很长，定期压缩可以节省 30-50% 的 token 开销。按小时计费的项目中，这直接影响你的利润。

---

### /clear

清空当前会话，从零开始。

```
/clear
```

**接单场景**：切换到同一项目的不同任务时使用。比如刚做完登录功能，要开始做支付功能，清空上下文避免干扰。

---

### /resume

恢复上一次中断的会话。

```
/resume
```

**接单场景**：下班前中断了工作，第二天回来直接恢复，不需要重新描述上下文。对于跨天的开发任务非常实用。

---

### /rewind

回退到对话中的某个节点，撤销后续的操作。

```
/rewind
```

**接单场景**：Claude Code 走错了方向，改了不该改的文件？用 /rewind 回到出错前的状态，重新给指令，避免手动还原。

---

## 代码质量

### /review

审查当前 Git 暂存区或工作区的代码改动。

```
/review
```

**接单场景**：每次提交代码前必做。帮你发现遗漏的边界情况、安全隐患、性能问题。交付给客户前的最后一道防线。

---

## 输出与导出

### /export

将当前对话导出为文件。

```
/export
```

**接单场景**：把和 Claude Code 的技术讨论导出，整理后作为技术文档交付给客户，或者留作项目交接材料。

---

## 模型与性能

### /model

切换当前使用的模型。

```
/model
```

**接单场景**：简单任务切换到更快更便宜的模型，复杂架构设计切换到更强的模型。灵活控制成本和质量的平衡。

---

### /fast

快速切换到更快的模型（通常是 Haiku）。

```
/fast
```

**接单场景**：做简单的格式调整、变量重命名等机械性工作时切换，速度更快、成本更低。

---

### /effort

调整模型的思考深度。

```
/effort high
/effort medium
/effort low
```

**接单场景**：架构设计用 high，日常编码用 medium，简单问答用 low。合理分配算力，优化成本结构。

---

## 上下文控制

### /context

管理对话中的上下文文件。

```
/context
```

**接单场景**：手动指定 Claude Code 需要关注的文件，避免它读取无关代码。在大型项目中能显著提升响应速度和准确性。

---

## 成本监控

### /cost

查看当前会话的 token 消耗和费用。

```
/cost
```

**接单场景**：实时监控开发成本。如果你按固定价格接单，这个命令帮你确保 API 费用不会吃掉利润。建议每隔一段时间就查看一次。

---

## 诊断工具

### /doctor

检查 Claude Code 的运行环境和配置是否正常。

```
/doctor
```

**接单场景**：遇到奇怪的报错或响应异常时，先跑一下诊断。快速排除环境问题，不浪费时间在配置上。

---

## 编辑模式

### /vim

切换到 Vim 风格的键位绑定。

```
/vim
```

**接单场景**：如果你习惯 Vim 操作，开启后在 Claude Code 中也能用熟悉的快捷键，提升输入效率。

---

## 命令使用频率参考

| 频率 | 命令 | 说明 |
|------|------|------|
| 每次启动 | `/init`, `/resume` | 初始化或恢复会话 |
| 每小时 | `/compact`, `/cost` | 控制上下文和成本 |
| 每次提交 | `/review` | 代码质量把关 |
| 按需使用 | `/model`, `/fast`, `/effort` | 调整模型策略 |
| 偶尔使用 | `/memory`, `/export`, `/doctor` | 项目管理和诊断 |
| 特殊场景 | `/clear`, `/rewind`, `/vim`, `/context` | 会话控制和偏好 |

## 小结

这 15 个命令不需要一次全记住。建议先熟练掌握 `/init`、`/compact`、`/review`、`/cost` 这四个核心命令，它们覆盖了自由职业开发中最关键的环节：项目初始化、成本控制、质量保障。其余命令在实际使用中逐步掌握即可。
