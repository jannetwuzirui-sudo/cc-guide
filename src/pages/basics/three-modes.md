# 三种模式详解

Claude Code 提供三种运行模式，适用于不同的工作场景。作为自由职业者，灵活切换模式能显著提升你的交付效率。

## 模式一：交互模式（Interactive Mode）

### 启动方式

```bash
claude
```

### 工作原理

启动后进入一个持续的对话界面，你可以连续输入指令，Claude Code 会保持上下文，理解你之前说过的内容。

### 自由职业场景

这是你最常用的模式，适合：

- **新项目探索**：刚拿到客户代码，需要边看边问
- **功能开发**：需要多轮对话逐步完善代码
- **调试排错**：反复定位问题，尝试不同方案

### 实战示例

```bash
cd ~/client-project
claude
```

```
> 帮我看看这个项目用了什么技术栈
> src/api 目录下的接口有哪些安全隐患
> 帮我修复刚才提到的 SQL 注入问题
> /review
```

每一轮对话都基于前面的上下文，不需要重复解释背景。

### 成本提示

交互模式下上下文会不断累积，token 消耗随对话增长。建议在对话较长时执行 `/compact` 压缩上下文，控制成本。

---

## 模式二：单次模式（One-shot Mode）

### 启动方式

```bash
claude -p "你的指令"
```

### 工作原理

执行一次指令后立即退出，不保留上下文。适合明确、独立的任务。

### 自由职业场景

适合批量处理和自动化任务：

- **快速代码审查**：客户发来一个文件，快速给出意见
- **生成模板**：一条命令生成标准化的项目文件
- **批量操作**：写脚本批量处理多个文件

### 实战示例

快速审查单个文件：

```bash
claude -p "审查 src/auth/login.ts 的安全性，列出所有潜在风险"
```

生成 API 文档：

```bash
claude -p "阅读 src/api/ 目录下所有路由文件，生成 API 文档，输出为 markdown 格式"
```

批量为多个项目生成 README：

```bash
for dir in ~/projects/client-*; do
  cd "$dir"
  claude -p "根据项目代码生成一份简洁的 README.md"
done
```

### 成本提示

单次模式每次调用都是独立的，没有上下文累积，token 消耗可预测，非常适合纳入自动化流程。

## 模式三：管道模式（Pipe Mode）

### 启动方式

```bash
echo "内容" | claude -p "指令"
```

或者：

```bash
cat file.txt | claude -p "指令"
```

### 工作原理

通过 Unix 管道将内容传给 Claude Code 处理，结果输出到标准输出。可以和其他命令行工具无缝串联。

### 自由职业场景

这是高级玩家的效率利器：

- **代码转换**：将一种格式批量转为另一种
- **日志分析**：快速从大量日志中提取关键信息
- **CI/CD 集成**：嵌入到自动化流水线中

### 实战示例

审查 Git 最近的改动：

```bash
git diff HEAD~3 | claude -p "审查这些代码改动，指出潜在问题"
```

分析错误日志：

```bash
cat error.log | claude -p "分析这些错误日志，归类问题并给出修复建议"
```

将 JSON 转为 TypeScript 类型定义：

```bash
cat api-response.json | claude -p "根据这个 JSON 生成 TypeScript 类型定义"
```

串联多个工具：

```bash
find src -name "*.ts" -exec cat {} \; | claude -p "找出所有未处理的 Promise rejection" > review-report.txt
```

### 成本提示

管道模式传入的内容会计入 token，处理大文件时注意控制输入大小，避免不必要的开销。

## 模式选择速查表

| 场景 | 推荐模式 | 原因 |
|------|----------|------|
| 新项目上手 | 交互模式 | 需要多轮探索和提问 |
| 功能开发 | 交互模式 | 需要持续上下文 |
| 快速审查 | 单次模式 | 一条命令搞定 |
| 批量处理 | 单次模式 | 可脚本化 |
| 日志分析 | 管道模式 | 直接传入内容 |
| CI/CD 集成 | 管道模式 | 无缝对接流水线 |

## 进阶建议

作为自由职业者，建议你：

1. **日常开发用交互模式**，保持对话流畅
2. **重复性任务用单次模式**，写成脚本复用
3. **数据处理用管道模式**，和现有工具链结合
4. 三种模式可以灵活组合，找到最适合你工作节奏的方式
