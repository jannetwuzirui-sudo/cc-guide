# 常见问题 FAQ

本页汇总了学员在使用 Claude Code、接单实战和部署上线过程中最常遇到的问题与解答。

---

## Claude Code 使用问题

### Q1: 如何获取和配置 API Key？

在 [Anthropic Console](https://console.anthropic.com/) 注册账号后，进入 API Keys 页面生成密钥。在终端中运行 `export ANTHROPIC_API_KEY=sk-ant-xxx` 或将其写入 `.bashrc` / `.zshrc` 文件中。注意不要将 API Key 提交到 Git 仓库。

### Q2: Token 费用大概是多少？怎么计算？

Claude API 按输入和输出 token 分别计费。一个中文汉字大约占 1-2 个 token。日常开发中，一次对话消耗几千到几万 token 不等，费用通常在几美分到几美元之间。可以在 Anthropic Console 的 Usage 页面查看实时用量。

### Q3: 应该选择哪个模型？

推荐使用 Claude Sonnet 进行日常开发，性价比最高。遇到复杂架构设计或疑难 bug 时，可以切换到 Claude Opus 获得更强的推理能力。简单的代码格式化或文本处理用 Haiku 即可。

### Q4: 上下文长度有限制吗？超出了怎么办？

Claude 模型有上下文窗口限制。当对话过长时，Claude Code 会自动进行上下文压缩。建议将大任务拆分成小步骤，每完成一个阶段就开启新对话，用 CLAUDE.md 文件记录项目上下文以便快速恢复。

### Q5: 网络连接不稳定怎么办？

国内用户可能遇到 API 连接超时的问题。可以尝试：1）使用稳定的网络环境；2）设置合理的超时时间；3）使用 API 代理服务。如果频繁断连，检查本地防火墙和代理设置。

### Q6: Claude Code 能做代码审查吗？

可以。直接将代码粘贴或通过文件路径让 Claude Code 阅读，然后要求它进行 code review。它会检查潜在 bug、性能问题、安全隐患和代码风格，并给出改进建议。建议在提交 PR 前用它做一轮审查。

### Q7: 如何让 Claude Code 同时编辑多个文件？

在提示词中明确说明需要修改的所有文件及其关联关系。例如："请同时更新 `api/routes.js` 中的路由和 `components/Form.vue` 中的表单，确保字段名一致。" Claude Code 会依次处理并保持文件间的一致性。

### Q8: 用 Claude Code 初始化新项目的最佳实践是什么？

先用一句话描述项目目标和技术栈，让 Claude Code 生成项目结构。然后逐步细化：先搭骨架，再填功能。建议在项目根目录创建 CLAUDE.md 写明技术选型、目录约定和编码规范，这样后续对话中 Claude Code 会自动遵循。

### Q9: 如何优化 API 调用成本？

几个实用技巧：1）提示词写清楚，减少来回对话；2）日常任务用 Sonnet 而非 Opus；3）善用 CLAUDE.md 减少重复说明上下文；4）将大任务拆成独立小任务；5）利用 `--output-format stream-json` 监控 token 消耗。

### Q10: 遇到报错信息看不懂怎么办？

直接把完整的错误信息发给 Claude Code，包括终端输出和相关代码片段。它会分析错误原因并给出修复方案。记得提供上下文，比如你在执行什么命令、使用什么框架版本，这样诊断更准确。

## 接单常见问题

### Q1: 客户压价怎么办？

首先明确自己的底线价格（时薪或项目最低价）。向客户展示你的作品集和效率优势——使用 Claude Code 你能更快交付高质量代码。可以提供阶梯报价：基础版、标准版、高级版，让客户自行选择。不要为了接单无底线降价。

### Q2: 项目做到一半客户不断加需求怎么办？

这就是"需求蔓延"（scope creep）。预防方法：开工前签书面协议，明确功能范围、交付物和修改次数。遇到新增需求时，礼貌告知属于额外工作，需要追加费用和时间。用文档记录每次需求变更。

### Q3: 客户需要域名备案（ICP备案）怎么处理？

ICP 备案是中国大陆网站的法律要求。备案流程通常需要 1-3 周，需要客户提供营业执照等材料。建议：1）提前告知客户备案周期；2）备案期间先用临时域名开发测试；3）推荐客户使用阿里云或腾讯云，备案流程较成熟。

### Q4: 客户不付尾款怎么办？

预防为主：采用分期付款（如 30% 预付 + 40% 中期 + 30% 验收），每个阶段交付前确认付款。合同中写明付款条款和违约责任。如果发生纠纷，先友好协商；协商不成可通过平台仲裁或法律途径解决。上线前不要交出源码和服务器权限。

### Q5: 如何管理项目时间线？

使用 Claude Code 的高效开发流程：1）需求分析阶段预留 1-2 天；2）开发周期按功能模块估算，乘以 1.5 作为缓冲；3）每周向客户同步进度；4）使用 Git 分支管理不同功能。给自己留余量，宁可提前交付也不要延期。

### Q6: 没有作品集怎么开始接单？

先做 3-5 个练手项目作为作品集：个人博客、企业官网、小程序、落地页等。用 Claude Code 可以快速完成这些项目。将作品部署上线，截图和源码整理好。也可以在开源社区贡献代码，积累技术影响力。

### Q7: 如何维护老客户关系获得复购？

交付后主动提供 1-2 周的免费维护期，及时响应小问题。定期回访，了解新需求。提供网站维护套餐（月费/年费）。节假日发送问候。老客户转介绍给予优惠。长期合作关系比一次性高价单更有价值。

### Q8: 接单需要注意哪些法律问题？

建议签订正式的开发服务合同，明确：项目范围、交付标准、付款方式、知识产权归属、保密条款、违约责任。个人接单收入需要依法纳税。如果金额较大，建议注册个体工商户或公司，开具正规发票。

## 部署常见问题

### Q1: 域名解析后访问不了，要等多久？

DNS 解析变更通常需要几分钟到 48 小时不等。国内 DNS 一般 10 分钟到 2 小时生效。可以用 `dig` 或 `nslookup` 命令检查解析状态。如果超过 24 小时仍未生效，检查 DNS 记录配置是否正确，或联系域名服务商。

### Q2: SSL 证书报错怎么解决？

常见原因：1）证书未正确安装或链不完整；2）证书已过期；3）域名与证书不匹配。解决方法：使用 Let's Encrypt 免费证书并配置自动续期；检查 Nginx/Apache 配置中证书路径是否正确；确保同时配置了中间证书。Vercel 和 Netlify 等平台会自动处理 SSL。

### Q3: 部署后页面显示 404 怎么办？

常见原因：1）构建输出目录配置错误（检查 `dist` 或 `build` 目录）；2）路由配置问题，SPA 应用需要配置 fallback 到 `index.html`；3）base path 未正确设置（如部署到子目录时）。检查部署平台的构建日志，确认文件确实被正确上传。

### Q4: 构建失败（Build Failed）怎么排查？

按以下步骤排查：1）先在本地运行 `npm run build` 确认能否成功构建；2）检查 Node.js 版本是否与部署平台一致；3）查看构建日志中的具体错误信息；4）确认 `package.json` 中依赖版本正确；5）清除缓存后重新部署。把错误日志发给 Claude Code 可以快速定位问题。

### Q5: 如何配置自定义域名？

一般流程：1）在域名服务商处添加 CNAME 记录指向部署平台提供的地址；2）在部署平台的项目设置中添加自定义域名；3）等待 DNS 生效和 SSL 证书签发。GitHub Pages 需要在仓库中添加 `CNAME` 文件。Vercel 和 Netlify 在控制台中操作即可。

### Q6: 网站访问速度慢怎么优化？

几个方向：1）开启 CDN 加速（国内推荐阿里云 CDN 或腾讯云 CDN）；2）优化图片（使用 WebP 格式、懒加载）；3）开启 Gzip/Brotli 压缩；4）减少 JavaScript 包体积（代码分割、tree-shaking）；5）使用 SSG 静态生成而非 SSR；6）合理设置浏览器缓存策略。
